# Brief: Write a Summary Report

*From: example-sender → example-agent | Date: 2026-01-01*

---

## What

Write a concise summary report (300–500 words) of the Q4 product metrics. Summarize key wins, misses, and one recommendation for Q1 prioritization. Output as a Markdown file.

## Why

The team lead needs a human-readable summary to share with stakeholders before the planning session on January 8. Pulling raw data from the dashboard is too noisy for that audience.

## Constraints

- Use only data from `context.md` — do not pull external sources
- Markdown output only (no HTML, no PDF)
- Write for a non-technical reader — no jargon, no acronyms unless defined
- Length: 300–500 words (not counting headers)

## Done When

- `result.md` contains the complete summary report
- `status.md` is updated to `status: done`
- The report is readable end-to-end without referencing other files
